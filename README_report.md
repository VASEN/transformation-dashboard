# process_report.py — обработка еженедельных отчётов

Скрипт читает еженедельный отчёт в формате Markdown, обновляет поле `current_status` в `data.json` и формирует готовое сообщение для Telegram.

## Использование

```bash
# Базовый запуск (обновляет data.json + формирует Telegram-файл)
python3 process_report.py ОТЧЕТ_01_04.md

# Явно указать data.json
python3 process_report.py ОТЧЕТ_01_04.md --data data.json

# Только сформировать Telegram-сообщение (data.json не трогать)
python3 process_report.py ОТЧЕТ_01_04.md --telegram-only

# Только обновить data.json (Telegram-файл не создавать)
python3 process_report.py ОТЧЕТ_01_04.md --no-telegram
```

## Что делает скрипт

1. Парсит блоки проектов из `.md`-файла (секции `✅ Выполнено` и `📍 Текущие этапы`).
2. Сопоставляет каждый проект с записью в `data.json`:
   - **по ID** — из ссылки `🔗 https://.../#/issues/XXXXX`;
   - **по имени** — нечёткое совпадение через `difflib` (порог 0.6), если ID не найден.
3. Записывает `current_status` в найденные проекты.
4. Создаёт резервную копию `data.json.bak` перед перезаписью.
5. Добавляет `summary.report_updated_at` с датой отчёта.
6. Сохраняет Telegram-сообщение в файл `telegram_ДД_ММ_ГГГГ.txt`.

## Формат входного файла

Каждый проект — блок из трёх строк-заголовков и двух опциональных секций:

```
🔵  Название проекта
👤  Ответственный
🔗  https://transformation.rm.mosreg.ru/#/issues/XXXXX

✅ Выполнено:
...текст...

📍  Текущие этапы (в работе):
...текст...
```

Секции `▬▬▬ 🔴 ПРИОРИТЕТНЫЕ ПРОЕКТЫ ▬▬▬` разделяют группы проектов.

## Зависимости

Только стандартная библиотека Python 3.8+. Сторонние пакеты не нужны.
