# CLAUDE.md — Трансформация Dashboard

Инструкция для Claude Code. Читать перед любой задачей по проекту.

---

## Проект

**Название:** Дашборд «Трансформация»  
**Назначение:** Мониторинг цифровой трансформации Комитета по конкурентной политике МО (ККП) и региональных центров трансформации (РЦТ).  
**Стек:** HTML + CSS + JS (`index.html`) + отдельный `data.json`, генерируемый Python-скриптом.  
**Деплой:** Amvera (amvera.ru) через Dockerfile → `nginx:1.25-alpine`.  
**Репозиторий:** `https://github.com/VASEN/transformation-dashboard`

---

## Архитектура

```
extract_data.py  →  data.json  →  index.html
(Python-скрипт)     (данные)      (дашборд, читает data.json через fetch)
```

**Файлы репозитория:**

| Файл | Назначение |
|------|-----------|
| `index.html` | Дашборд — единственный UI-файл |
| `data.json` | Данные — коммитится в git |
| `extract_data.py` | Скрипт генерации data.json из Excel |
| `Dockerfile` | `FROM nginx:1.25-alpine` |
| `amvera.yml` | Конфиг деплоя Amvera |
| `.gitignore` | Excel-файлы исходников в git не попадают |

**Принцип обновления данных:**
1. Положить актуальные `.xlsx` рядом с `extract_data.py`
2. Запустить `python3 extract_data.py`
3. Закоммитить обновлённый `data.json` и запушить → Amvera подхватит

---

## Три вкладки дашборда

1. **Обзор** (`screen-exec`) — KPI-карточки, SVG-gauges по кураторам, тепловая карта дедлайнов, таблица проектов. Фильтры: Год, Ответственный.
2. **Детализация** (`screen-detail`) — карточка выбранного проекта: цель, показатели, команда, трудозатраты, задачи.
3. **Задачи** (`screen-tasks`) — таблица задач с фильтрами по статусу, проекту, исполнителю.

Excel-файлы в git **не хранятся** (в `.gitignore`). Передаются вручную при обновлении.

### `issues_1.xlsx` — Redmine-экспорт (508 строк × 173 колонки)

| Трекер | Кол-во | Роль |
|--------|--------|------|
| `Мероприятие проекта` | 470 | задачи/этапы |
| `Паспорт проекта` | 37 | карточки проектов |
| `Карточка ведомства` | 1 | **игнорировать** |

**Ключевые исходные поля:**
```
#                    — Redmine ID
Проект               — название проекта
Трекер               — тип записи
Статус               — Новая / В работе / Закрыта / Выполнено / На проверке
Тема                 — заголовок задачи/проекта
Назначена            — руководитель проекта (Паспорт) / исполнитель (Мероприятие)
Ответственный        — владелец проекта (держатель) — только в Паспорте
Срок завершения      — дедлайн
Готовность           — % выполнения (0–100)
Дата начала
РЦТ                  — принадлежность к РЦТ
ЦИО проекта          — ведомство-исполнитель
Куратор от Мингос    — NULL везде → НЕ использовать
```

**Правило куратора:** поле `Куратор от Мингос` пустое во всех записях. Куратор определяется по фамилии в поле `Назначена`:
- `Кренёва` → Кренёва АА
- `Родевальд` → Родевальд СЕ
- `Гуляев` → Гуляев ВА

Строки с `Проект = NaN` — итоги по кураторам, порядок:
`Комитет и РЦТ → Кренёва АА → Родевальд СЕ → Гуляев ВА`

## Поля объектов в JS

### project (из `data.projects`)

```js
p.id               // Redmine ID
p.name             // название проекта
p.status           // 'В работе' | 'Закрыта' | ...
p.owner            // полное имя держателя
p.owner_short      // 'Кренева А.А.'
p.manager          // полное имя руководителя
p.manager_short    // 'Иванов И.И.'
p.deadline         // 'DD.MM.YYYY'
p.start_date       // 'DD.MM.YYYY'
p.pct              // 0–100
p.project_type     // 'Межведомственный' | 'Внутриведомственный'
p.goal             // текст цели
p.indicators       // текст показателей
p.team             // текст команды
p.problem          // текст проблемы
p.current_status   // текущий статус (может быть null)
p.plan_hours       // план трудозатрат, часы
p.fact_hours       // факт трудозатрат (может быть null)
p.plan_units       // план высвобождения шт.ед.
p.fact_units       // факт высвобождения (может быть null)
```

### task (из `data.all_tasks`)

```js
t.id               // Redmine ID
t.project          // название проекта
t.theme            // заголовок задачи
t.status           // 'Новая' | 'В работе' | 'Закрыта' | ...
t.executor         // полное имя исполнителя
t.executor_short   // 'Иванов И.И.'
t.deadline         // 'DD.MM.YYYY'
t.urgency          // 'overdue' | 'today' | 'urgent' | 'soon' | 'ok'
t.pct              // 0–100
t.parent_id        // ID родительской задачи
```

### curator (из `data.curators`)

```js
c.name             // 'Кренёва АА'
c.plan_units       // план высвобождения (из штатки)
c.fact_units       // факт высвобождения (из НМА)
c.vysv_pct         // % = fact_units / plan_units * 100
c.staff_total      // кол-во фактических ставок
c.vacancies        // вакансии
```

## Бизнес-логика

### Формула liberation gauge

```
vysv_pct = высвобождение_шт.ед. (NaN-строка из НМА) ÷ план_20% (из штатки) × 100
```

**Подтверждённые результаты:**

| Куратор | % |
|---------|---|
| Комитет и РЦТ | 128% |
| Кренёва АА | 130% |
| Родевальд СЕ | 144% |
| Гуляев ВА | 89% |

### Фильтры вкладки Обзор

- **Год** — фиксирован 2026 (статичное поле, не менять)
- **Ответственный** — dropdown по `p.owner_short` из `data.projects`
- KPI-карточки пересчитываются при смене фильтра
- Блок gauges **не** фильтруется
- Выбранный ответственный → gauge `opacity: 1`, остальные → `opacity: 0.35`

### Срочность задач (urgency)

```
overdue  — дедлайн прошёл
today    — дедлайн сегодня
urgent   — дедлайн ≤ 5 дней
soon     — дедлайн ≤ 14 дней
ok       — остальные
```

## SVG gauges — важно

Использовать `stroke-dasharray` на полукруговых path. Расчёт через arc endpoints ломается.

```js
// Полуокружность: длина дуги = π * r
const arcLen = Math.PI * radius;
const filled = arcLen * Math.min(percent, 100) / 100;
// stroke-dasharray: `${filled} ${arcLen - filled}`
// Путь: M cx-r,cy  A r r 0 0 1 cx+r,cy
```

## Частые ошибки — не повторять

| Ошибка | Правило |
|--------|---------|
| `d['Ответственный']` в JS | Поле называется `p.owner` / `p.owner_short` |
| `d['Назначена']` в JS | Поле называется `p.manager` / `p.manager_short` |
| `data.project_details` | Использовать `data.projects` |
| Куратор из `Куратор от Мингос` | NULL везде — куратор берётся из `Назначена` по фамилии |
| arc endpoint calculation в SVG | Использовать `stroke-dasharray` |
| Inline JSON в HTML | Нет — архитектура fetch-based, данные в `data.json` |
| GitHub Actions deploy | Заброшен, деплой только через Amvera |
| xlsx в git | Только `data.json` — исходники в `.gitignore` |
