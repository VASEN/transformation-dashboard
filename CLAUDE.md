## Архитектура

```
transformation-dashboard.html   ← единственный deliverable
Dockerfile                       ← FROM nginx:1.25-alpine, COPY html → /usr/share/nginx/html/index.html
amvera.yml                       ← конфиг деплоя Amvera
```

**Принцип:** всё в одном HTML-файле. Данные из Excel → Python-скрипт → inline JSON `const DATA = {...}`. Никогда не использовать `fetch('data.json')` или любые внешние файлы — это ломается на статик-хостинге.

**Четыре вкладки:**
1. **Обзор** — KPI-карточки, SVG-gauges по кураторам, тепловая карта, таблица проектов
2. **Проекты** — список проектов с фильтрами
3. **Задачи** — задачи/мероприятия с фильтрами
4. **Штат** — таблица штатного расписания

- `Кренёва АА` → фамилия «Кренёва»
- `Родевальд СЕ` → фамилия «Родевальд»
- `Гуляев ВА` → фамилия «Гуляев»

## Бизнес-логика

### Формула liberation gauge

```
% высвобождения = высвобождение_шт.ед. (из НМА, NaN-строка) ÷ план_20% (из штатки) × 100
```

Подтверждённые результаты:
- Комитет и РЦТ: **128%**
- Кренёва АА: **130%**
- Родевальд СЕ: **144%**
- Гуляев ВА: **89%**

### Фильтры Overview-вкладки

- **ГОД** — по году `Срок завершения` паспортов (дефолт: 2026, варианты: 2025, Все)
- **ОТВЕТСТВЕННЫЙ** — уникальные значения `Ответственный` у паспортов, фильтруется по ГОД
- KPI-карточки пересчитываются при изменении фильтров
- Блок liberation gauges **не** фильтруется по году
- Gauge выбранного ответственного — 100% opacity, остальные — 0.35

### Статусы задач

```
Закрыта    — 234  (завершено)
Новая      — 170  (не начата)
В работе   — 102  (в процессе)
Выполнено  —   1
На проверке —  1
```

## SVG gauges — важно

Использовать `stroke-dasharray` на полукруговых path. Расчёт через arc endpoints ломается. Проверенный паттерн:

```js
// Полуокружность: длина дуги = π * r
const arcLen = Math.PI * radius;
const filled = arcLen * (percent / 100);
// stroke-dasharray: `${filled} ${arcLen}`
// stroke-dashoffset: 0
// Путь: M (left) A r r 0 0 1 (right)
```

## Воспроизводство данных (Python)

Скрипт пересборки inline JSON:

```python
import pandas as pd, json

# 1. Читаем три файла
issues = pd.read_excel('issues_1.xlsx')
staffing = pd.read_excel('ШТАТКА_ДБ.xlsx')
liberation = pd.read_excel('ПРОЕКТЫ_Данные_по_высвобождению.xlsx')

# 2. Паспорта и мероприятия
passports = issues[issues['Трекер'] == 'Паспорт проекта'].copy()
tasks     = issues[issues['Трекер'] == 'Мероприятие проекта'].copy()

# 3. Определяем куратора из поля 'Назначена'
def get_curator(assigned):
    if pd.isna(assigned): return None
    for name in ['Кренёва', 'Родевальд', 'Гуляев']:
        if name in str(assigned):
            return name
    return None

passports['curator'] = passports['Назначена'].apply(get_curator)

# 4. Embed в HTML
data_json = json.dumps({
    'projects': passports[['#','Проект','Тема','Назначена','Ответственный',
                            'Статус','Срок завершения','Готовность','curator']].to_dict('records'),
    'tasks':    tasks[['#','Проект','Тема','Назначена','Статус',
                        'Срок завершения','Готовность']].to_dict('records'),
    'staffing': staffing.to_dict('records'),
    'liberation': liberation.to_dict('records'),
}, ensure_ascii=False, default=str)
```

## Частые ошибки — не повторять

| Ошибка | Правило |
|--------|---------|
| `fetch('data.json')` → 404 | Только inline JSON в HTML |
| `d.person` / `d.owner_short` | Реальные имена полей — см. раздел выше |
| `data.project_details` | Использовать `data.projects` |
| `Куратор от Мингос` | NULL везде, не использовать |
| arc endpoint calculation в SVG | Использовать stroke-dasharray |
| GitHub Actions deploy | Заброшен, деплой только через Amvera |
